01
Patient support Production-ready

Grounded support assistant — RAG that can't invent device instructions

The problem

A diabetes-device companion app generates thousands of repetitive, safety-sensitive support questions. Support teams can't scale with them — and a chatbot that invents medical-device instructions isn't a UX bug, it's a patient-safety incident.

What I shipped

A grounded RAG service on AWS Bedrock: hybrid retrieval (vector + BM25) with Cohere reranking, generation locked to approved documentation — verbatim citations, novel numbers rejected by a hallucination filter. Query-aware model routing, context compression and prompt caching cut latency and cost; an inline relevancy judge rejects bad context with zero extra LLM calls. PII detection, prompt-injection blocking and out-of-scope refusal guard the edges — and every deploy is gated on a 129-question golden-set eval harness.

−173 ms end-to-end P50  ·  −19% cost per query  ·  92% test coverage  ·  eval-gated deploys

ClinicSentry — compliance middleware for clinical AI agents

An open-source Python library that sits between any agent framework — LangGraph, CrewAI, OpenAI Agents SDK, Claude SDK, MCP — and the outside world, enforcing clinical guardrails without touching agent code. A PHI firewall with adversarial normalization (homoglyphs, zero-width characters, base64 payloads), a risk-tiered escalation router that fails closed, and a tamper-evident HMAC audit chain, mapped clause-by-clause to HIPAA, FDA TPLC, IEC 62304 and the EU AI Act. Backed by a separate benchmark harness: six benchmark families covering detection F1, adversarial robustness, latency and compliance correctness.

~0.7 ms p95 PHI scan on 3 KB notes  ·  zero LLM calls on the hot path  ·  4 regulatory frameworks auto-checked

02
Research 2026 · in prep

StressBench — when agents pass the parser but fail the task

An evaluation harness that stress-tests LLM agents under injected faults and reports two numbers side by side: strict_pass (the answer matched the format and tool-call protocol) and verified (an action-metamorphic verifier recomputed the true end-state from the tool outputs the agent actually saw). The gap between them — format-without-correctness — is the headline metric. Compares ReAct, native function-calling, calibration and verifier-in-the-loop remediation agents, with bootstrap CIs, McNemar + Holm correction, and calibration error.

16 verifier task families  ·  9 injected fault types  ·  dual strict/verified reporting  ·  targeting ICMLT 2026
